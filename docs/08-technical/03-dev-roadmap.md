# 開發路線圖

## 25.1 開發階段

### Phase 0：前期準備（2 週）

- [ ] 確認企劃書所有系統設計
- [ ] 建立Godot專案框架（資料夾、Autoload、基本場景）
- [ ] 設計並實作幾何圖形繪製工具（DrawShape類）
- [ ] 建立UI主題（geometric_theme.tres）
- [ ] 建立資料管理框架（DataManager + JSON載入）

### Phase 1：核心戰鬥系統（4 週）

- [ ] 實作戰鬥場景基礎框架
- [ ] 實作回合管理（TurnManager）
- [ ] 實作傷害計算（DamageCalculator）
- [ ] 實作技能系統基礎（施放 + 效果）
- [ ] 實作元素相剋倍率
- [ ] 實作狀態異常系統
- [ ] 實作戰鬥UI（技能選擇、道具使用、血條等）
- [ ] 實作基本敵人AI
- [ ] 實作戰鬥結果（經驗/掉落）
- [ ] 測試：完成 5 場不同戰鬥

### Phase 2：探索系統（3 週）

- [ ] 實作節點地圖生成演算法
- [ ] 實作節點地圖UI（繪製 + 互動）
- [ ] 實作各類節點（戰鬥/事件/寶箱/商人/中繼/祭壇）
- [ ] 實作事件系統（載入JSON + 選項 + 結果）
- [ ] 實作樓層進度管理
- [ ] 測試：完成 3 層探索流程

### Phase 3：安全區與城鎮（3 週）

- [ ] 實作安全區場景框架
- [ ] 實作商店系統（買/賣）
- [ ] 實作鍛造工坊（強化裝備）
- [ ] 實作圖書館（技能研究）
- [ ] 實作使魔小屋（培養/進化）
- [ ] 實作酒館（任務板）
- [ ] 實作傳送系統
- [ ] 實作旅館/存檔
- [ ] 測試：完整的安全區循環

### Phase 4：角色系統（2 週）

- [ ] 實作升級系統（EXP + 屬性成長）
- [ ] 實作裝備系統（穿戴/效果計算）
- [ ] 實作飾品系統（被動效果觸發）
- [ ] 實作技能學習/升級/突變
- [ ] 實作被動技能
- [ ] 實作稱號系統
- [ ] 測試：Lv.1~20 角色成長曲線

### Phase 5：使魔系統（2 週）

- [ ] 實作使魔核心獲取/孵化
- [ ] 實作使魔戰鬥AI（4種模式）
- [ ] 實作使魔成長/餵養
- [ ] 實作使魔進化
- [ ] 實作使魔指令切換
- [ ] 測試：完整的使魔培養循環

### Phase 6：內容填充（4 週）

- [ ] 製作第一區（1F~9F）完整內容
- [ ] 製作第二區（10F~19F）完整內容
- [ ] 製作第三~五區內容
- [ ] 製作第六~十區內容
- [ ] 製作所有Boss
- [ ] 製作所有NPC對話/任務
- [ ] 填充技能/道具/飾品資料庫

### Phase 7：打磨與平衡（3 週）

- [ ] 數值平衡調整（全樓層通測）
- [ ] UI動畫與過場特效
- [ ] 音效/BGM整合
- [ ] 粒子特效製作

### Phase 8：測試與發布（2 週）

- [ ] 完整流程測試（1F~100F）

## 25.2 里程碑目標

| 里程碑 | 時間點 | 可驗收內容 |
|--------|--------|----------|
| M1 | Phase 1 完成 | 可進行一場完整戰鬥 |
| M2 | Phase 2 完成 | 可探索 3 層地牢 |
| M3 | Phase 3 完成 | 可使用城鎮所有功能 |
| M4 | Phase 4+5 完成 | 完整的角色+使魔成長循環 |
| M5 | Phase 6 完成 | 可遊玩前 50 層 |
| M6 | Phase 7 完成 | 全內容可遊玩 |
| M7 | Phase 8 完成 | 正式發布版 |

## 25.3 預計總工期

**約 25 週（約 6 個月）**

> 備註：以單人開發為前提估算。若有額外人力可大幅縮短內容填充階段。
