# UI/UX 設計規範

## 20.1 設計原則

- **幾何美學**：所有UI元素使用幾何圖形構成，乾淨俐落
- **色彩語言**：元素顏色統一貫穿（火=紅、水=藍、雷=黃...）
- **資訊層級**：重要資訊突出，次要資訊可展開
- **點擊優先**：所有操作都能透過滑鼠點擊完成
- **動畫節制**：過場動畫簡短精緻，不拖慢節奏
- **響應回饋**：每個操作都有視覺或音效回饋

## 20.3 HUD 元素

戰鬥中常駐顯示的 HUD：

| 元素 | 位置 | 內容 |
|------|------|------|
| 主角HP條 | 左下 | 綠色血條 + 數值 |
| 主角MP條 | 左下（HP下方） | 藍色魔力條 + 數值 |
| 使魔HP條 | 左下（MP下方） | 使魔名稱 + 血條 |
| 敵人HP條 | 上方 | 每個敵人獨立血條 |
| 回合指示 | 右上 | 當前行動順序列表 |
| 狀態圖示 | 角色旁 | buff/debuff 小圖示 |
| 傷害數字 | 浮動 | 傷害/回復數字飄出 |
| 屬性提示 | 浮動 | 「效果絕佳！」「效果不佳...」 |

## 20.4 選單系統

主選單（按 ESC 或點擊選單按鈕）：

```
┌── 選單 ──────────┐
│  [狀態]           │
│  [背包]           │
│  [技能]           │
│  [使魔]           │
│  [任務]           │
│  [圖鑑]           │
│  [設定]           │
│  [存檔]           │
│  [返回標題]        │
└──────────────────┘
```

## 20.5 通知與提示系統

| 通知類型 | 顯示方式 | 持續時間 |
|---------|---------|---------|
| 道具獲得 | 右上浮動 | 2秒 |
| 升級 | 畫面中央+特效 | 3秒 |
| 任務完成 | 左側滑入 | 3秒 |
| 技能領悟 | 畫面中央+特效 | 4秒 |
| 成就達成 | 上方橫幅 | 3秒 |
| 新區域到達 | 全畫面+區域名 | 2秒 |
| 使魔進化 | 演出動畫 | 5秒 |
