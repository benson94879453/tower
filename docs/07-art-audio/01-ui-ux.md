# UI/UX 設計規範

## 20.1 設計原則

- **幾何美學**：所有UI元素使用幾何圖形構成，乾淨俐落
- **色彩語言**：元素顏色統一貫穿（火=紅、水=藍、雷=黃...）
- **資訊層級**：重要資訊突出，次要資訊可展開
- **點擊優先**：所有操作都能透過滑鼠點擊完成
- **動畫節制**：過場動畫簡短精緻，不拖慢節奏
- **響應回饋**：每個操作都有視覺或音效回饋

## 20.3 HUD 元素

戰鬥中常駐顯示的 HUD：

| 元素 | 位置 | 內容 |
|------|------|------|
| 主角HP條 | 左下 | 綠色血條 + 數值 |
| 主角MP條 | 左下（HP下方） | 藍色魔力條 + 數值 |
| 使魔HP條 | 左下（MP下方） | 使魔名稱 + 血條 |
| 敵人HP條 | 上方 | 每個敵人獨立血條 |
| 回合指示 | 右上 | 當前行動順序列表 |
| 狀態圖示 | 角色旁 | buff/debuff 小圖示 |
| 傷害數字 | 浮動 | 傷害/回復數字飄出 |
| 屬性提示 | 浮動 | 「效果絕佳！」「效果不佳...」 |

## 20.6 UI 組件規格表

| 組件名稱 | 視覺特徵 | 交互行為 | 適用場景 |
| :--- | :--- | :--- | :--- |
| **主面板 (Main Panel)** | 粗邊框, 深色半透明背景, 幾何直角/微圓角 | 頂層顯示, 遮蔽背景, 提供關閉按鈕 | 角色狀態, 技能樹, 設定選單 |
| **子面板 (Sub Panel)** | 細邊框, 單色背景, 內嵌於主面板 | 邏輯分組, 滾動支持 | 道具清單, 技能列表, 屬性詳細數據 |
| **行動按鈕 (Action Button)** | 實色填充, 粗邊框, 文字居中, 點擊縮放 | 點擊觸發立即執行, 懸停高亮 | 戰鬥指令, 確認按鈕, 進入下一層 |
| **選擇按鈕 (Selection Button)** | 鏤空邊框, 點擊後變色/填充, 文字左對齊 | 切換選中狀態, 單選/多選 | 選單項目, 技能選擇格, 裝備位選擇 |
| **資源條 (Resource Bar)** | 分段填充, 顏色區分 (HP-綠, MP-藍, EXP-黃) | 數值變動時平滑過渡, 暴擊/受傷時閃爍 | 角色血量, 魔力, 經驗值, 敵人生命 |
| **道具格 (Item Slot)** | 正方形, 邊框根據品質著色, 右下角顯示數量 | 點擊打開詳情, 拖拽移動(若支持) | 背包, 裝備欄, 商店庫存 |
| **對話框 (Dialog Box)** | 底部文本區 + 上方人物圖示/名稱 | 逐字顯示文本, 點擊-下一步, 選擇-觸發分支 | 事件觸發, NPC 對話, 劇情揭露 |
| **狀態圖示 (Status Icon)** | 小圓形/方形, 邊框顏色對應屬性, 內部簡約符號 | 懸停顯示狀態名稱與效果 | buff/debuff 顯示, 屬性抗性標記 |
| **工具提示 (Tooltip)** | 小尺寸深色面板, 白色文字, 指向性小三角 | 隨鼠標移動, 延遲 0.2s 出現 | 查看屬性細節, 道具說明, 技能效果 |

## 20.7 排版與佈局規範

| 項目 | 規範內容 | 備註 |
| :--- | :--- | :--- |
| **邊距 (Margin)** | 全域統一 24px $\rightarrow$ 48px | 確保畫面呼吸感 |
| **內邊距 (Padding)** | 組件內部統一 8px $\rightarrow$ 16px | 避免文字貼邊 |
| **邊框粗細 (Border)** | 主邊框: 4px / 子邊框: 2px | 強調層級感 |
| **對齊規則** | 標題: 左對齊 $\rightarrow$ 數據: 右對齊 $\rightarrow$ 按鈕: 居中 | 遵循視覺掃描路徑 |
| **字體大小** | 標題: 24pt / 正文: 16pt / 數值: 18pt (粗體) / 備註: 12pt | 建立資訊層級 |
| **色彩映射** | 火-紅 / 水-藍 / 雷-黃 / 風-綠 / 土-棕 / 光-白 / 暗-紫 / 無-灰 | 貫穿全遊戲的屬性語言 |
